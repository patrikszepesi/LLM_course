Positional Encodings:

Because transformers process the entire sequence in parallel, they lack inherent information about the positions of tokens. Positional encodings are added to the token embeddings to provide this positional information.

How Positional Encodings Work

Positional encodings are vectors added to the token embeddings.
These vectors are created using sinusoidal functions of different frequencies.

Example for the sentence “I am a robot”:

item Token Embeddings: [EI][E_{\text{I}}][EI​], [Eam][E_{\text{am}}][Eam​], [Ea][E_{\text{a}}][Ea​], [Erobot][E_{\text{robot}}][Erobot​]
item Positional Encodings: [P0][P_0][P0​], [P1][P_1][P1​], [P2][P_2][P2​], [P3][P_3][P3​]

Item Combined Embeddings:

[[EI+P0],[Eam+P1],[Ea+P2],[Erobot+P3][[E_{\text{I}} + P_0], [E_{\text{am}} + P_1], [E_{\text{a}} + P_2], [E_{\text{robot}} + P_3][[EI​+P0​],[Eam​+P1​],[Ea​+P2​],[Erobot​+P3​]

Positional Encoding Calculation

Detailed calculations for positions 0 (“I”) and 1 (“am”) using the specified formulas for positional encoding, assuming (d=4) and using (n=100) for simplification:

For even indices (i.e., 0, 2, 4, …), the value is computed using the sine function:
PE(pos, 2i)=sin⁡(pos1002id) PE_{\text{(pos, 2i)}} = \sin\left(\frac{\text{pos}}{100^{\frac{2i}{d}}}\right) PE(pos, 2i)​=sin(100d2i​pos​)

For odd indices (i.e., 1, 3, 5, …), the value is computed using the cosine function:
PE(pos, 2i+1)=cos⁡(pos1002id) PE_{\text{(pos, 2i+1)}} = \cos\left(\frac{\text{pos}}{100^{\frac{2i}{d}}}\right) PE(pos, 2i+1)​=cos(100d2i​pos​)

Why Use Sine and Cosine Functions for Positional Encodings?

  1. Smooth and Continuous: Sine and cosine waves create smooth, gradual changes in values. This is helpful for models to understand the relative positions of words – “am” is closer to “I” than it is to “robot.”

  2. Repeating Patterns: Sine and cosine waves have repeating patterns. This allows the model to handle sentences of varying lengths. Even if a sentence is longer than any it has seen before, the pattern of the positional encoding can be extended.

How It Works in Practice

For the word “am”:

Now, the word “am” has a unique set of numbers (0.84, 0.54, 0.1, 1.0) that represent its position. No other word in any sentence will have this exact combination.

The example I gave with the word “am” was a simplified illustration to explain the concept of using sine and cosine for positional encoding.

In reality, modern transformer models use much higher dimensional positional encodings. Instead of just four values. This ensures that even if we get to the part of the sine or cosine wave that got repeated we will have a combination of these encodings which makes it impossible for the model to mix up which word is first

Why So Many Dimensions?

The high dimensionality of positional encodings serves several purposes:

  1. Uniqueness: Even if one or two wave patterns repeat, the combination of many different frequencies virtually eliminates the chance of two words having the same encoding, regardless of their position in the sentence.

  2. Expressiveness: With more dimensions, the model can capture subtle nuances of word order, such as the difference between “a big red ball” and “a red big ball.”

  3. Robustness: High-dimensional encodings are less prone to interference or confusion when dealing with long and complex sentences.

In essence, the multi-dimensional nature of positional encodings is like a fingerprint for each word’s position within a sentence. This ensures the model can accurately distinguish between words and understand the meaning of the sentence as a whole.

The periodicity (repeating pattern) of sine and cosine waves offers several advantages for positional encoding in language models:

  1. Overview:
  • The Problem: Language models need to handle sentences of varying lengths. If we used a simple linear encoding (1 for first word, 2 for second, etc.), the model might struggle with sentences longer than it has seen during training.
  • The Solution: Since sine and cosine waves repeat, the pattern of positional encoding can be easily extended. Even if a sentence has 1000 words, the model can continue the wave pattern to create unique codes for each position.
  1. Relative Positional Information:
  • The Problem: In language, the relative position of words is often more important than the absolute position. For example, understanding that “am” comes right after “I” is crucial, whether it’s the 2nd and 3rd word in a sentence or the 52nd and 53rd.
  • The Solution: The periodic nature of the waves provides information about relative positions. The difference in values between the sine/cosine encodings of two words gives the model a sense of how far apart those words are in the sentence.
